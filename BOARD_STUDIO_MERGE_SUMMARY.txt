KEY FINDINGS
------------

1. ARCHITECTURAL DIFFERENCES

   Old Version:
   - BoardStudioPage → Multiple Contexts → PatternRenderer (direct)
   - No abstraction layer between orchestration and rendering
   - Risk of bypasses

   New Version:
   - BoardStudio → BoardStudioContext → FrameRenderer → PatternRenderer
   - FrameRenderer provides abstraction and protection
   - Bypasses architecturally prevented

2. PATTERNRENDERER INTEGRATION

   Old Version:
   - Calls PatternRenderer directly (lines 1947, 2023)
   - Passes mode correctly
   - No bypasses found (good!)
   - But no protection against future bypasses

   New Version:
   - All rendering through FrameRenderer wrapper
   - FrameRenderer adds editing chrome
   - PatternRenderer owns content rendering
   - Bypasses architecturally impossible

3. CRITICAL ISSUES TO FIX

   Issue 1: Mode Contract Inconsistency
   - Old uses: 'studio' | 'preview' | 'assist'
   - New uses: 'studio' | 'preview'
   - PatternRenderer checks: 'edit' || 'studio'
   - FIX: Standardize to 'studio' | 'preview' | 'assist'

   Issue 2: Props Data Format
   - V0 stores: array format
   - PatternRenderer expects: object OR array
   - Database stores: object format
   - FIX: Always use array, add normalization layer

   Issue 3: Role-Based Special Cases
   - Cover frame has hardcoded special handling
   - Violates pattern-based rendering principle
   - MediaUploader only works for cover frames
   - FIX: Pattern-based with prop detection

   Issue 4: Type Inconsistency
   - StudioFrame (v0)
   - FrameInstance (database)
   - FrameData (PatternRenderer)
   - FIX: UnifiedFrame as single source of truth

ARCHITECTURAL DECISIONS
-----------------------

1. PatternRenderer as Design Agent
   - Single authoritative presentation layer
   - All rendering flows through it
   - No bypasses allowed

2. FrameRenderer Abstraction
   - Protects PatternRenderer contract
   - Handles editing chrome
   - Manages mode switching

3. Props Always Arrays
   - Never objects in business logic
   - Adapters handle storage format
   - Consistent, ordered, composable

4. Pattern-Based Rendering
   - Render based on pattern, not role
   - Role is metadata only
   - Patterns are reusable

5. Three Modes
   - studio: Full editing
   - preview: Final presentation
   - assist: AI-assisted editing
